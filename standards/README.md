## 開発標準

この組織で採用する、言語を問わない共通の開発標準を定める。

### 型安全性

- 厳格な型チェックを必須とする
- any 型や型チェックの除外は原則禁止とし、やむを得ない場合は理由をコメントで残す

### コードフォーマットとリント

- フォーマッターおよびリンターを導入し、手動でのスタイル調整を行わない
- pre-commit フックにより、コミット時に自動でフォーマットとリントを実行する
- CI でもフォーマットとリントのチェックを行い、違反があればマージをブロックする

### テスト

- すべてのリポジトリにテストフレームワークを導入する
- テストは CI で自動実行し、失敗時はマージをブロックする

### コードスタイル

- 行長は 100 文字を上限とする
- 各言語標準の命名規約に従う

### 依存関係の管理

- 本番依存と開発依存を明確に分離する
- 開発用のツール（テストフレームワーク、リンター等）を本番依存に含めない

### ブランチ戦略

パッケージマネージャー（PyPI、npm 等）で配布するリポジトリは [GitHub Flow](https://docs.github.com/en/get-started/using-github/github-flow) を採用する。

### バージョニング

パッケージマネージャーで配布するリポジトリは [セマンティックバージョニング](https://semver.org/lang/ja/) を採用する。
